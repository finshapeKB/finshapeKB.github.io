Products
This section describes all products that represent Finshape's value proposition. Value Proposition is a primary component of the business model. This is a description of the goods or services that a company offers and why they are desirable to customers or clients.
Digital banking building blocks
Finshape’s main product is a custom Digital Banking solution. “Custom” in this context refers to a specific configuration of the provided building blocks (functional capabilities) combined with developing front-end applications and selected add-ons from scratch. These building blocks are not standalone products but rather DBOS infrastructure elements, DBOS components, connectors (APIs), tools, and applications from which the final product can be assembled.

Workspaces
Workspace is a desktop or mobile page that is opened as bank's employees or clients log in to their banking application. Workspaces are usually designed for a specific Role to provide handling Role specific Tasks and their steps.
Workspaces for Bank's Employee (WAC)
Bank's Employee workspace is a desktop or a mobile page that is opened as bank's employees log in to their banking application. Bank's Employee manage their banking operations using Roles, so each role may have it's own workspace.
Workspaces for Corporate Client (Business)
Bank's Corporate Client workspace is a desktop or a mobile page that is opened as bank's corporate clients log in to their internet banking application. Corporate clients manage their banking operations using Roles, such as bookkeeper, CEO, CFO etc., so each role may have it's own workspace.
Workspaces for Private Client (Retail)
Workspace for Private Client is a desktop or a mobile UI starting page that is designed for Private Client role. The workspace is opened as Private Clients login to the internet banking application. Private Clients manage their banking operations within workspace.

Tasks
Task is a specific result that may be reached by completing some steps or actions.
Task types
Task types were defined to ease understanding their business nature. From the business point of view we may divide all possible tasks into separate domains: Functional Tasks and Product tasks. Tasks are divided into types within each domain and may be divided into subtypes.
Product tasks: PT
Functional tasks: FT
Task Steps: TS
We consider for function (step) any combination of command, operators and data set that aim to get one specific result. Combination of functions (steps) form business task.
Step types
Functional task's Steps: FTS
Product task's Steps: PTS

Data and their Sources
This section may be useful for developers that are responsible for connecting DBOS to Core Banking systems, Third party systems and internal DBOS component connections, as well as DBOS configuration. To handle business tasks and run functions you may need data from various sources. Data sources effects DBOS configuration.
Data List from all sources
The List of data from all sources provides all input data that are needed to handle business tasks an functions to run.
Data Sources List
Data sources list provides all sources that provide input data that are needed to handle business tasks an functions to run.
DBOS source components
DBOS component produces new data by running its own or third party functions. So, DBOS component needs input data and provides its output data and functions. DBOS component may provide data and functions to other DBOS components, front-end applications and third party systems.
Core banking source systems
Core banking systems provides data and functions to DBOS components.
Third party source systems
Third party systems provide data and functions to DBOS components.
