Glossary
Products
This section is divided into two subsections and outlines all products that make up Finshape’s value proposition. A value proposition describes the goods and services a company offers and explains why they are valuable to customers. Alongside the product descriptions, the section also includes a technical subsection Product building blocks that details elements and components Finshape’s products consist of.
Product's building blocks
Finshape’s main product is a custom Digital Banking solution. Custom in this context refers to a specific configuration of the provided building blocks (functional capabilities) combined with developing front-end applications and component extensions. These building blocks are not standalone products but rather DBOS infrastructure elements, DBOS components, connectors (APIs), tools, and applications from which the final product can be assembled.
Products
We offer to our clients three standalone products (solutions), some add-ons to the Core Digital Banking solution, and services. Standalone means that product may be delivered to a client on Project level and doesn't depend on other DBOS components.

Roles
Enterprise role is a defined set of tasks, responsibilities, and activities that a person is expected to handle within the organization. It specifies what the individual does, how they contribute to business processes, and the scope of their responsibilities across the enterprise.

Workspaces
Workspace is a desktop or mobile page that is opened as bank's employees or clients log in to their banking application. Workspace may be designed for all Roles at the same time, or for a specific Role to provide handling only Role specific Tasks. Workspace usually comprise Introduction or Overview GUI page.
Workspaces for Bank's Employee (WAC)
Bank's Employee workspace is a desktop page that is opened as bank's employees log in to their banking application. Bank's Employee manage their banking operations using Roles, so each role may have it's own workspace.
Workspaces for Corporate Client (Business)
Bank's Corporate Client workspace is a desktop or a mobile page that is opened as bank's corporate clients log in to their internet banking application. Corporate clients manage their banking operations using Roles, such as bookkeeper, CEO, CFO etc., so each role may have it's own workspace.
Workspaces for Private Client (Retail)
Workspace for Private Client is a desktop or a mobile UI starting page that is designed for Private Client role. The workspace is opened as Private Clients login to the internet banking application. Private Clients manage their banking operations within workspace.
Tasks
Task is a set of steps that have to be done to reach a specific result.
Task types
Task types were defined to ease understanding their business nature. From the business point of view we may divide all possible tasks into separate domains: Functional Tasks and Product tasks. Tasks are divided into types within each domain and may be divided into subtypes.
Product tasks
Product Tasks are tasks which Finshape handle to design, produce, sale, deliver and putting its products to use. Product domain tasks are divided into types. Product task types are associated with enterprise processes. Each task type may have subtypes.
Functional tasks
Functional Tasks domain comprises all banking tasks which bank employees and their clients handle using DBOS. Moreover this domain comprises all tasks that are need to be done to install and maintenance DBOS and front-end applications. Functional domain tasks are divided into seven types, each task type may have subtypes. Example, Functional tasks/Business tasks/Payment tasks/...
Task Steps
Task step is one simple action that in combination with other actions leads to the task completion.

Data sources
This section may be useful for developers that are responsible for connecting DBOS to Core Banking systems, Third party systems and internal DBOS component connections, as well as DBOS configuration. To handle business tasks and run functions you may need data from various sources. Data sources effects DBOS configuration.
Data List from all sources
The List of data from all sources provides all input data that are needed to handle business tasks an functions to run.
Data Sources List
Data sources list provides all sources that provide input data that are needed to handle business tasks an functions to run.
DBOS source components
DBOS component produces new data by running its own or third party functions. So, DBOS component needs input data and provides its output data and functions. DBOS component may provide data and functions to other DBOS components, front-end applications and third party systems.
Core banking source systems
Core banking systems provides data and functions to DBOS components.
Third party source systems
