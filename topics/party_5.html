    • Finshape products

      This section describes all products that are being delivered to the client. Here we describe products from the business point of view. Value Proposition is a primary component of the business model. This is a description of the goods or services that a company offers and why they are desirable to customers or clients.

      • Authentication and Authorization solution

        The authentication and authorization solution is a comprehensive, secure, and scalable service that provides identity, access, and operational governance across the entire digital banking system. This service is delivered through the DBOS components Digital Identity (GAAS) and GEN (Entitlements).

      • Login Application

        Login Application is a React-based web application implements the authentication GUI. DBOS solutions include two Login Applications: one for bank clients and another for bank employees.

      • Start Digital Banking solution

        The Start Digital Banking solution provides the essential capabilities required to deliver core digital banking services, including secure customer operations, seamless integration with existing bank systems, and support for additional banking functionalities. These functionalities include storing and providing data, executing transactions, and performing product-related actions, all supported through a unified integration layer. The product includes the Authentication and Authorization solution along with other key components: Instant Data (GDS), Payments and Instructions (TPM), Integration Hub (IHUB), and Service Proxy (SPX).

      • Admin Console (WAC)

        A React-based administrative web application for bank employees, designed to configure the DBOS Entitlements (authorization) model and manage user security elements.

      • Desktop Internet Banking application

        Internet banking application is a web application implementing the internet banking GUI. It is a React-based web application, implementing complete GUI logic (rendering bank data, processing user request flows, certification (signing), etc.), communicating with server components only via data REST and/or GraphQL APIs. Internet Banking implements a GUI for on-behalf access, where a bank employee operates the solution on behalf of a selected user.

      • iOS and Android Smartphone Banking

        Mobile applications for iOS (iPhone) and Android smartphones implement a GUI for smartphone banking. These applications are based on the DBOS Journey Designer platform, leveraging React Native and Expo frameworks for cross-platform development and delivery. Part of the applications will be also specific user journey implementing a GUI of digital onboarding process.

      • Four Eyes

        The Four Eyes component is an optional add-on designed for corporate banking clients who require stronger internal control. It ensures that sensitive administrative actions are only executed after being reviewed and approved by a second authorized user, providing an extra layer of security and governance.

      • Notification Centre (GNC)

        GNC is an optional add-on that manages and generates notifications for selected events, such as balance changes or specific account movements.

      • Distribution (GDC)

        GDC is an optional add-on that handles the delivery of notifications to clients, ensuring that alerts and messages generated by the bankâ€™s systems are reliably sent through the configured communication channels.

      • Exchange rates (XCR)

        XCR is an optional add-on that provides exchange rate information for foreign currency operations and supports currency-dependent business rules, such as currency limits or thresholds.

      • Experience builder (EXB)

        EXB is an optional add-on that stores user-specific data for web or mobile applications, such as personalized dashboard settings or other custom user preferences.

      • Document Generation Service

        The Document Generation Service is an optional add-on that enables the automated creation of various banking documents, such as contracts, statements, and confirmations. It ensures that documents are produced accurately, consistently, and in a fully digital form, supporting both customer-facing and internal processes. The service consists of two components: Print Server (GPS), which handles the actual document generation, and SIDER, a technical extension that allows large or complex documents to finish processing in the background even after the user logs out.

      • PSD2 Support Service

        PSD2 Support Service is an optional add-on that delivers the mandatory functionality required for PSD2 compliance within the EU. It enables secure access for licensed third-party providers, manages customer consents, and ensures proper registration and handling of third-party entities. The service consists of three core components: OpenBank for exposing PSD2-compliant APIs, ConsentReg for managing and storing customer consent, and TpReg for registering and maintaining third-party provider information.

      • Product Support

        Product support is a service consists of: onboarding for banking employees etc.

      • Onboarding application

        Application implementing a GUI for the digital onboarding process. This application uses the same technology approach as iOS and Android smartphone banking applications - Journey Designer (React Native).

      • Maintenance service

        Maintenance service consists of services related to products' updates, health checkups etc.
