    This section is divided into two subsections and outlines all products that make up Finshape’s value proposition.   3. Digital banking building blocks

    Finshape’s main product is a custom Digital Banking solution. Custom in this context refers to a specific configuration of the provided building blocks (functional capabilities) combined with developing front-end applications and component extensions. These building blocks are not standalone products but rather DBOS infrastructure elements, DBOS components, connectors (APIs), tools, and applications from which the final product can be assembled.

  4. Applications

    An application is a software package that performs a specific function directly for an end user. An application can be self-contained or a group of programs. The program is a set of back-end operations that runs the application for the user. Applications use the computer's operating system (OS) and other supporting programs, typically system software, to function. An application requests services from and communicates with other technologies via an application programming interface (API).

  5. Desktop Internet Banking application

    Desktop Internet banking application is an optional add-on to the Core Digital Banking solution. It's a web application implementing the internet banking GUI where a bank employee operates the solution on behalf of a selected user.

      • DBOS infrastructure elements

        DBOS infrastructure elements are required for deploying and ensuring the smooth operation of the front-end and back-end on the client side, including all key services, environments, and technical resources that enable the system to run and function.

      • Connectors (API)

        Connectors are elements for connecting composite application with other third party applications and internal composite application components between each other.

      • DBOS service components

        DBOS component is the container for program that implements the business logic and configuration information. The program is a set of back-end operations that the application runs for the user. A component offers a service to other components and in turn consumes functions offered by other services using service-oriented interfaces.
